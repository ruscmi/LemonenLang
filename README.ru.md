Всем привет! 

Lmnlang (Lemonenlang) — это быстрый скриптовый язык. 

Что там сделано? 
- **Лексер** 
- **Парсер** 
- **AST-дерево** 
- **Интерпретатор** 
- **Раннер**

Я использую unordered_map в памяти для переменных, 
он работает очень быстро и почти мгновенно строит AST-дерево. 
Язык обновляется ежедневно, и я работаю над этим самостоятельно. 

**Установка и компиляция.**

== **ДЛЯ LINUX** == 
Чтобы начать, скопируйте репозиторий и перейдите к нему. 

```bash 
git clone https://github.com/ruscmi/LemonenLang && cd LemonenLang
```

После этого просто запустите bash-скрипт, 
который скомпилирует всё необходимое для вас. 

```bash 
chmod +x builder.sh
./builder.sh
```
== **ДЛЯ WINDOWS** == 
Для Windows придётся потеть. 
Сначала установите установщик GCC и используйте 
его для компиляции проекта. 
Но я бы посоветовал использовать WSL (терминал Linux на Windows). 

**Выполнение файлов и REPL** 
Для перехода в режим REPL (режим чтения цикла печати) 
Вам нужно зайти в каталог, где был скомпилирован код: 
```bash 
cd LemonenLang 
```
Если у вас есть проект в другом каталоге, запишите путь к нему: 
НА WINDOWS 
```bash 
cd путь\к\директории 
``` 
НА LINUX 
```bash 
cd путь/к/директории 
``` 
Далее, когда вы окажетесь внутри LemonenLang, просто напишите: 
НА WINDOWS (powershell, к чёрту cmd) 
```bash
.\build\lmnlang.exe 
``` 
НА LINUX
```bash
./build/lmnlang 
```
будет работать ТОЛЬКО если файлы скомпилированы, 
Для запуска файлов используйте тот же принцип, 
но запишите путь к файлу и укажите флаг --file: 
НА LINUX 

```bash
./build/lmnlang --file путь/к/директории 
``` 
НА WINDOWS 

```bash
.\build\lmnlang.exe --file path\to\directory 
``` 
чтобы просмотреть руководства, вы можете ввести следующее внутри REPL: 

```lmnlang
man list
```
